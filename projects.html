Real-Time Boston Bus Tracker

Catch that Boston Bus!

I developed this tracker in Week 5 of MIT xPRO Bootcamp, May, 2022. I wrote async JS functions to deploy real-time data.

Github Demo

Here I pulled maps from mapbox.com and real-time data from the MBTA (Massachusetts Bay Transportation Authority) to show the location of one of Boston's Route 1 buses. I did this using an access token and async functions in Javascript. The user decides whether and when to start tracking the bus by clicking on a button. The tracking marker shows on the map immediately after that click. But my code fetches the real-time location of the bus prior to rendering the map and prior to the user's decision, in order to pre-center the map on the bus's exact current coordinates. This avoids any unnecessary movement of the map, providing the user with a smooth visual experience.

Responsive Eyes

Everybody likes to be noticed.

This April, 2022 project bound DOM elements to mouse events. Even a little interactivity can be delightful!

Github Demo

The starter code for this bootcamp project created one eye. I added a second eye. (Although, a cyclops version could also be fun.) I used JavaScript to give positions to the pupils of the eyes. The benefit of this exercise was learning how to tie the position of the pupils to the position of the user's cursor and how--through the user's mouse -- to enable the user to control the direction of the gaze. Well-articulated CSS properties are key to depicting the eyes' movement.

Pac-Men Factory

My first project.

I developed this project in Week 4 of MIT xPRO Bootcamp, April, 2022. I used JavaScript to change DOM elements.

Github Demo

The coin-operated Pac-Man video game rocketed to fame soon after its 1980 debut. The code I'm presenting here uses the Pac-Man character to explore the DOM. Building on starter code from MIT, I wrote JavaScript functions to set an initial, randomized position and velocity for each of our iconic yellow friends on the screen and to ensure that he bounces from walls upon impact. The chomping motion is rendered by toggling between two images--one with mouth open and the other with mouth shut. Extra clicks on the "Add PacMan" button add more PacMen to the scene.
